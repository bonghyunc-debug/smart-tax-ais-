# 프로젝트 구조 및 세법 로직 1차 정리

## 1. 저장소 구조 요약
- **프론트엔드 구성**: Vite 기반 React 단일 페이지 앱. 진입점 `App.tsx`에서 상태 관리(useReducer)와 주요 컴포넌트 렌더링을 담당.
- **주요 디렉터리/파일**
  - `components/`: 신고서 입력 폼, 결과 요약, 모달 등 UI 모듈 집합.
  - `utils/taxCalculations.ts`: 양도소득세 핵심 계산 로직(세율, 공제, 가산세, 기한 계산 등) 정의.
  - `types.ts`: 상태와 계산 결과에 사용되는 타입 정의.

## 2. 현재 설계된 신고 유형 추론
- 자산 유형이 "일반주택", "1세대1주택_고가주택", "토지", "분양권", "미등기" 등으로 분류되고, 보유기간·장기보유특별공제·비사업용 토지 중과·농어촌특별세 등이 반영됨. 이는 **부동산·분양권 양도소득세(양도소득세) 신고**를 목표로 설계된 것으로 해석된다. 종합소득세나 부가가치세 등 다른 세목 요소는 등장하지 않음.
